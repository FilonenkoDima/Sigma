Для подальшої успішної підтримки програми, необхідно додати абстракцію.

Необхідно створити абстрактний клас/інтерфейс для класу "Склад" та для класу "Продукт".
Програмам постійно розвивається і додаються нові продукти, для спрощення реалізації
нових продуктів необхідна абстракція. Нові продукти можуть мати специфічну умовну зберігання,
тому для них прийдеться створювати новий вид складу, для цього потрібні абстракція складів.

До існуючої абстракції, потрібно створити сервіси, які також будуть мати свою абстракцію.
Для складу необхідно мати методи, які повертають, наявність продуктів у складу, 
різноманітні методи парсу(з файлу, з масиву, зі стрічки), запис у файл та індексацію.
Для продуктів необхідно мати методи, які дають можливість змінювати ціну, парсити зі стрічки,
отримувати інформацію у вигляді стрічки, та метод Equals для подалбших можливих операцій.

Також необхідно створити окремий клас, який буде відповідати за лог помилки, запис їх у файл
та їх вивведення на екран.
